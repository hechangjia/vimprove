# v2.6 Daily Vim Mastery Design

## Purpose

Vimprove already covers core motions, operators, Visual mode, macros/registers, marks/jumps, and real-world editor setup. v2.6 should close the largest remaining gap between the browser tutorial and daily Vim use: command-line mode, practical substitute workflows, and focused drills beyond `hjkl`.

## Recommended Scope

Build a small, shippable v2.6 release with three connected parts:

1. **Command-line mode foundation**
   - Add `:` command-line entry from normal mode.
   - Support `Escape` to cancel and `Enter` to execute.
   - Implement practical non-file commands first: `:s/old/new/`, `:s/old/new/g`, `:%s/old/new/g`, `:q`, `:w`, `:wq`, `:q!`.
   - Treat file commands as simulated status commands inside the browser; they should update state metadata but not touch the local filesystem.

2. **Chapter 11: Daily Editing Workflows**
   - Lessons should teach Ex command-line basics, substitute on one line, substitute across a buffer, and a first-week workflow review.
   - Keep examples concrete: rename text, fix repeated log labels, and recover from common mistakes.

3. **Find Target minigame**
   - Add a drill that shows a target character in a dense line and scores the user for reaching it with `f/F/t/T/;/,` instead of repeated `h/l`.
   - Use this to cover a different skill than Snake and 2048: line-local precision.

## Excluded From v2.6

- Full file IO, filesystem-backed `:w`, or real quit/navigation changes.
- Full Ex parser.
- Ranges beyond current line and whole buffer.
- Regex replacement flags beyond `g`.
- Visual Block insert/replay and system clipboard registers; these remain future engine work.

## Architecture

Extend `VimState` with command-line fields: `commandLine`, `commandStatus`, and optional last Ex command metadata. The reducer should route keys to a small `exCommands.ts` module while `mode === 'command'`. The command parser should be deliberately narrow and deterministic so tests can define exact behavior.

Lessons continue to use the existing `Lesson` data model. The minigame should use a new `find-target` content block and a React component in `src/components/minigame`, following the existing `hjkl-snake` and `game-2048` patterns.

## Success Criteria

- Engine tests cover command-line entry/cancel/execute and substitute variants.
- Translation structure tests pass for all new Chapter 11 lessons.
- Lesson registry test updates the published lesson count intentionally.
- `npm run typecheck`, `npm run lint`, focused tests, quickcheck, and build pass.
