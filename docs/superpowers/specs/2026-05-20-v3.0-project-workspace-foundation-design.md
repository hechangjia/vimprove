# v3.0 Project Workspace Foundation Design

## Purpose

v2.x built a broad Vim command curriculum. v3.0 should change the product level: from learning commands in one buffer to completing project-style editing tasks across multiple files.

The first major-version slice should establish the core workspace loop: search across files, collect results, jump through them, and edit the active file.

## Recommended Scope

1. **Quickfix engine foundation**
   - Add `quickfixList`, `quickfixIndex`, and `quickfixOpen` to `VimState`.
   - Support narrow literal `:vimgrep /pattern/`.
   - Support `:cnext`, `:cprev`, `:copen`, and `:cclose`.
   - Quickfix jumps switch simulated buffers, load the target file, and place the cursor at the match.

2. **Workspace UI signal**
   - Extend `VimChallenge` to show the active buffer name when multi-buffer state exists.
   - Show a compact quickfix panel when `quickfixOpen` is true.
   - Keep this as a compact project panel, not a full IDE layout rewrite.

3. **Chapter 14: Project Workspace**
   - Add four lessons: workspace mental model, project search, quickfix navigation, and multi-file cleanup review.
   - Use `initialBuffers` to present multi-file project exercises.

## Excluded From v3.0

- Real filesystem IO.
- Full regex `:vimgrep`.
- Quickfix locations from compiler/LSP.
- Full multi-pane workspace rendering or tree editing.
- `:cfdo`, `:argdo`, `:bufdo`, and quickfix bulk commands.

## Architecture

Keep quickfix inside the pure `VimState` and `exCommands.ts`. Quickfix entries reference simulated buffer indexes, so all behavior remains deterministic and testable. UI changes stay in `VimChallenge`, because project workflow lessons already use challenge state.

## Success Criteria

- Tests cover `:vimgrep`, quickfix open/close, and next/previous quickfix jumps across buffers.
- Quickfix panel displays in challenges when opened.
- Chapter 14 adds four translated lessons.
- Version moves to `3.0.0`.
- `npm run typecheck`, `npm run lint`, focused tests, quickcheck, and build pass.
