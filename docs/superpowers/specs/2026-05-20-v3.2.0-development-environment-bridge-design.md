# v3.2.0 Development Environment Bridge — Design Spec

**Goal:** Turn Vimprove from a Vim command tutorial into a bridge toward modern development-environment workflows.

## Scope

This release is content-first. It adds Chapter 16 with four lessons that connect Vim skills to editor choice, Vim mode in IDEs, LSP/AI assistance, and remote development. It does not add LSP, AI, terminal, or container execution to the browser engine.

## Chapter 16 Lessons

1. `dev-environment-mental-model` — explain editor, shell, language server, formatter, tests, AI assistant, and remote runtime as a system.
2. `vim-mode-everywhere` — teach how to carry Vim mode into VS Code, JetBrains, shells, commit messages, and browser text areas.
3. `lsp-ai-vim-workflow` — clarify where Vim motion/operator thinking complements completion, inline chat, rename, references, and diagnostics.
4. `remote-dev-workflow-review` — provide a realistic remote/dev-container style editing checklist and a browser challenge using existing Vim commands.

## UI Integration

The homepage learning paths gain a fourth card, “Development Environment”, linking to the first Chapter 16 lesson. Existing lesson routing remains unchanged.

## Test Strategy

- Update lesson registry expected count to 80 lessons and 16 categories.
- Use existing translation structure/completeness tests to enforce zh and zh-lively content.
- Run typecheck, lint, quickcheck, and build before reporting completion.
