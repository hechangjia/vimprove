# v2.8 Daily Workflow Polish Design

## Purpose

Vimprove now has broad Vim coverage, but the product layer still has a few gaps that make it feel less polished than the engine and curriculum: the support matrix is stale, learning paths are not visible enough, and daily batch-edit workflows stop at basic substitute.

v2.8 should make the existing 61-lesson curriculum easier to enter and more useful for daily editing.

## Recommended Scope

Build a focused polish release with four parts:

1. **Accurate Vim status matrix**
   - Update Settings -> Vim Status so supported features from v2.2-v2.7 are no longer listed as unsupported.
   - Add categories for Visual, Macros/Registers, Marks/Jumps, Command-line, and Project Navigation.

2. **Visible learning paths**
   - Add route cards to the home page for 30-minute survival, 7-day daily driver, and project navigation.
   - Route cards should jump to concrete starting lessons using existing `LESSONS` slugs.
   - Keep this as navigation, not a new progress system.

3. **Daily Ex command upgrades**
   - Extend substitute to support line ranges such as `:1,3s/old/new/g`.
   - Add a narrow `:g/pattern/s/old/new/g` global command for literal pattern matching plus literal substitute.
   - Keep parsing deterministic; do not introduce a full regex engine or full Ex grammar.

4. **Operator Gym minigame**
   - Add a small repeated drill for choosing semantic edits (`dw`, `ci"`, `dap`, `di(`).
   - This complements existing movement games by training operator + motion/text-object decisions.

## Excluded From v2.8

- Full Ex ranges (`.+3`, marks, visual range), full regex replacement, confirmation prompts, and capture-group replacement.
- A new spaced-repetition scheduler or persistent learning-path completion model.
- Full multi-pane editor rendering.
- Full Neovim parity expansion for new Ex commands; the browser Ex subset remains intentionally narrow.

## Architecture

The status matrix stays inside `src/components/settings/VimStatusTab.tsx`. Learning paths should be a small data list in `HomePage` or a focused data helper if the component grows. Ex command upgrades remain in `src/core/exCommands.ts` with tests in `src/core/exCommands.test.ts`. Operator Gym should follow the existing minigame content-block pattern with a new `operator-gym` block type and component in `src/components/minigame`.

## Success Criteria

- Settings no longer marks Visual, Registers, Marks, Macros, Command-line, or Project Navigation as unsupported.
- Home page shows three practical learning paths and clicking them starts the intended lesson.
- Tests cover line-range substitute and global substitute.
- Operator Gym renders from a lesson content block.
- `npm run typecheck`, `npm run lint`, focused tests, quickcheck, and build pass.
