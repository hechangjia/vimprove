# v2.9 Navigation Review Polish Design

## Purpose

Vimprove covers character, word, file, paragraph, search, and project navigation. The remaining foundational navigation gap is screen-level movement: scrolling by half/full pages, placing the cursor line in the viewport, and jumping to visible top/middle/bottom lines.

v2.9 should teach these commands as daily reading/navigation skills without rebuilding the editor into a full terminal viewport emulator.

## Recommended Scope

Build a focused v2.9 release with three parts:

1. **Screen navigation engine slice**
   - Add lightweight viewport metadata to `VimState`: `viewportTop`, `viewportHeight`, and `pendingZ`.
   - Support `Ctrl-d`, `Ctrl-u`, `Ctrl-f`, `Ctrl-b` as half/full page movement.
   - Support `zz`, `zt`, `zb` as viewport placement commands.
   - Support `H`, `M`, `L` as visible-screen line jumps.

2. **Chapter 13: Screen Navigation**
   - Add four lessons covering half/full page movement, viewport placement, screen line jumps, and a navigation review.
   - Keep challenges concrete and browser-safe. They should validate cursor and viewport state, not real DOM scroll position.

3. **Scroll Surfer minigame**
   - Add a minigame that drills `Ctrl-d/u/f/b` and `zz/zt/zb`.
   - This complements Snake/2048/Find Target/Window Navigator/Operator Gym by training screen-scale movement rather than grid movement or semantic edits.

## Excluded From v2.9

- Pixel-perfect terminal scroll emulation.
- Real DOM scroll synchronization inside `VimChallenge`.
- `gj` / `gk` wrapped-line navigation.
- Scrolloff, smooth scrolling, folds, tabs, and quickfix.

## Architecture

Keep screen navigation in `vimReducer.ts` as normal-mode commands. Store viewport state in `VimState` so challenges and tests can validate it deterministically. `state.buffer` remains the canonical active text. Add a new `scroll-surfer` content block and component under `src/components/minigame`.

## Success Criteria

- Unit tests cover `Ctrl-d/u/f/b`, `zz/zt/zb`, and `H/M/L`.
- Chapter 13 has four lessons and matching zh / zh-lively translation structure.
- Scroll Surfer renders from a lesson content block.
- `npm run typecheck`, `npm run lint`, focused tests, quickcheck, and build pass.
