# v2.3.0 Productivity Core Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the productivity-core Vim features from the roadmap: macros, named registers, marks, jumplist/changelist, and Chapters 8-9.

**Architecture:** Extend the pure reducer state with explicit pending states instead of parsing raw strings outside the reducer. Implement in small vertical slices: registers/macros first, then marks/jumplist/changelist, then lessons and version/docs. Keep system clipboard registers as documented placeholders in content until an async UI bridge exists.

**Tech Stack:** TypeScript core reducer, Vitest, React lesson data, i18next lesson translations.
