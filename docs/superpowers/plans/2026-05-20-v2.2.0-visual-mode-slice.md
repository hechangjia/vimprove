# v2.2.0 Visual Mode Slice Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [x]`) syntax for tracking.

**Goal:** Add the first vertical slice of Chapter 7: Visual char, Visual-Line, and Visual-Block mode entry, motion, exit, editor selection highlighting, and `d` / `y` / `c`.

**Architecture:** Extend the Vim state machine with `mode: 'visual' | 'visual-line' | 'visual-block'` and `visualAnchor`. In this slice, `v` / `V` / `Ctrl-v` enter visual modes, motions move the cursor while preserving the anchor, `Escape` / the same visual key exits, and visual selections can be yanked, deleted, or changed. Blockwise paste and exact Neovim block insert replay are intentionally deferred until the register model can carry register kind metadata.

**Tech Stack:** TypeScript core reducer, Vitest, React editor renderers, Tailwind/CSS editor classes, i18next challenge namespace.

### Task 2: Implement Visual State Foundation

**Files:**
- Modify: `src/core/types.ts`
- Modify: `src/core/stateUtils.ts`
- Modify: `src/core/vimReducer.ts`

- [x] **Step 1: Extend types**

Change `Mode`:


Add to `VimState`:

```ts

```

- [x] **Step 2: Preserve snapshots**

In `createSnapshot`, copy `visualAnchor`.

In `clearPendingStates`, set `visualAnchor: null`.

- [x] **Step 3: Implement visual key handling**

Add a `handleVisualKey` function:
- `Escape` or `v`: return to normal and clear count / anchor.
- Motions `h j k l w b e W B E 0 $ ^ _ gg G { } %`: move cursor with existing motion logic and preserve anchor.
- Operators `y`, `d`, `c`: convert the inclusive visual selection into an end-exclusive range and apply charwise yank/delete/change.
- In visual-line mode, convert the selected lines into a linewise range.
- In visual-block mode, convert the selected lines and columns into a rectangle and operate on each line.
- Other keys: leave state unchanged for this slice.

In normal mode, `v` enters visual mode with `visualAnchor: { ...cursor }`; `V` enters visual-line mode at column 0; `Ctrl-v` enters visual-block mode.

### Task 3: Add Editor Highlighting

**Files:**
- Modify: `src/components/challenge/VimChallenge.tsx`
- Modify: `src/components/settings/VimPlaygroundTab.tsx`
- Modify: `src/index.css`

- [x] **Step 1: Add selection check**

For each rendered character, consider it selected when:
- `state.mode === 'visual'`
- or `state.mode === 'visual-line'`
- or `state.mode === 'visual-block'`
- `state.visualAnchor` exists
- The character position is between `visualAnchor` and `cursor`, inclusive.
For visual-line mode, every rendered character on selected lines is selected.
For visual-block mode, characters inside the selected line/column rectangle are selected.

- [x] **Step 2: Add class**
